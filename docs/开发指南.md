## 一、快速开始（第一次配置）

### 1.1 克隆项目

```bash
git clone <仓库地址>
cd ShangHaiFoodData
```

### 1.2 配置文件

```bash
cp .env .env
cp config/config.example.yaml config/config.yaml
```

编辑 `.env`，填入共享 MySQL 服务器信息：

```env
MYSQL_HOST=<共享服务器IP>
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=<密码>
MYSQL_DATABASE=shanghai_food
APP_SECRET_KEY=your-secret-key-change-this
```

### 1.3 初始化数据库

在共享 MySQL 服务器上执行一次（由负责后端的同学操作）：

```bash
mysql -h <服务器IP> -u root -p < scripts/init_db.sql
```

## 二、前端开发

### 环境要求

- Node.js >= 18
- npm >= 9

### 启动

```bash
cd frontend
npm install
npm run dev   # 启动在 http://localhost:3000
```

Vite 已配置代理，`/api` 请求自动转发到 `http://localhost:8000`。

### 目录结构

```
frontend/src/
├── pages/
│   ├── Login/          # 登录页
│   ├── Dashboard/      # 数据大屏
│   └── Search/         # 餐厅搜索
├── components/
│   ├── charts/         # ECharts 图表组件
│   └── common/         # 通用组件
├── api/                # Axios 请求封装
├── store/              # Zustand 状态管理
└── utils/              # 工具函数
```

### 前端开发时后端还没好怎么办？

在 `frontend/src/api/` 目录下，每个接口文件顶部加一个 mock 开关：

```js
const USE_MOCK = true  // 后端好了改成 false

export const getOverview = async () => {
  if (USE_MOCK) return mockOverviewData
  return axios.get('/api/v1/analysis/overview')
}
```

mock 数据放在 `frontend/src/mock/` 目录下。

### ECharts 上海地图

上海地图 JSON 文件需要单独下载：

```bash
# 在 frontend/public/ 目录下放置
# 下载地址：https://geo.datav.aliyun.com/areas_v3/bound/310000_full.json
# 文件名保存为：shanghai.json
```

使用方式：

```js
import * as echarts from 'echarts'
import shanghaiJson from '/shanghai.json'

echarts.registerMap('shanghai', shanghaiJson)
```

## 三、后端开发

### 环境要求

- Python >= 3.10
- pip

### 启动

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

API 文档自动生成：http://localhost:8000/docs

### 目录结构

```
backend/app/
├── main.py             # FastAPI 入口
├── core/
│   ├── security.py     # JWT + BCrypt
│   ├── deps.py         # 依赖注入（获取当前用户）
│   └── config.py       # 读取 .env 配置
├── models/             # SQLAlchemy ORM 模型
├── schemas/            # Pydantic 请求/响应模型
├── api/v1/
│   ├── auth.py         # 登录接口
│   ├── restaurants.py  # 餐厅搜索接口
│   └── analysis.py     # 分析接口
└── services/           # 业务逻辑
```

### 接口开发规范

所有接口统一返回格式：

```python
from fastapi.responses import JSONResponse

def success(data):
    return {"code": 200, "message": "success", "data": data}

def error(msg, code=400):
    return JSONResponse({"code": code, "message": msg, "data": None}, status_code=code)
```

### 数据库连接

```python
# backend/app/core/config.py 读取 .env
DATABASE_URL = f"mysql+pymysql://{MYSQL_USER}:{MYSQL_PASSWORD}@{MYSQL_HOST}/{MYSQL_DATABASE}"
```

## 四、爬虫开发

### 环境要求

- Python >= 3.10
- Chrome 浏览器（Selenium 需要）
- ChromeDriver（版本需与 Chrome 匹配）

### 启动

```bash
cd crawler
pip install -r requirements.txt
scrapy crawl meituan_spider
```

### 爬虫策略

1. 先爬美团上海餐厅列表页
2. 获取：餐厅名、地址、菜系、均价、评分、区域
3. 写入 MySQL `restaurant_info` 表
4. 如果遇到验证码/封禁，立即停止，切换备用方案

### 备用方案：导入公开数据集

```bash
# 下载数据集后，运行导入脚本
python scripts/import_dataset.py --file <数据集路径>
```

导入脚本需要做字段映射和数据清洗（见 `scripts/import_dataset.py`）。

### 数据清洗要求

- 电话脱敏：`138****1234`
- 区域标准化：统一为 16 个区名
- 菜系归类：参考 `config/dictionaries/cuisine_dict.py`
- 去重：按 `name + address` 去重

## 六、常见问题

### Q: 前端请求后端报 CORS 错误？

在 `backend/app/main.py` 确认有：

```python
from fastapi.middleware.cors import CORSMiddleware
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])
```

### Q: MySQL 连接失败？

检查 `.env` 中的 `MYSQL_HOST` 是否填写了共享服务器 IP，不是 `localhost`。

### Q: ECharts 地图不显示？

确认 `frontend/public/shanghai.json` 文件存在，且 `echarts.registerMap('shanghai', ...)` 在图表初始化之前调用。

### Q: JWT token 过期？

前端在 Axios 拦截器中处理 401 响应，自动跳转登录页：

```js
axios.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(err)
  }
)
```

### Q: 爬虫被封了怎么办？

立即停止，不要继续尝试。切换到备用数据集方案，通知组长。
