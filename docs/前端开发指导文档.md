## 二、技术约定

### 文件规范
- 页面组件放 `views/`，可复用组件放 `components/`
- 所有文件用 `.jsx` 扩展名
- 路径别名：`@` 代表 `src/`，例如 `import { request } from '@/utils/request'`

### 接口调用规范
所有接口统一通过 `src/utils/request.js` 的封装调用，**不要直接用 axios**：

```js
// src/api/analysis.js 示例
import { get } from '@/utils/request'

const USE_MOCK = true  // 后端未就绪时改为 true，联调时改为 false

const mockOverview = { total_count: 3000, avg_rating: 4.2, avg_price: 85.5 }

export const getOverview = () => {
  if (USE_MOCK) return Promise.resolve({ code: 200, data: mockOverview })
  return get('/analysis/overview')
}
```

**重要**：每个 api 文件顶部都要有 `USE_MOCK` 开关，后端没好之前用 mock 数据开发。

### 后端接口响应格式
所有接口返回统一格式，取数据时用 `res.data`：
```js
// 后端返回：{ code: 200, message: "success", data: {...} }
const res = await getOverview()
const data = res.data  // 取这里
```

### 状态管理
用 Zustand，store 在 `src/store/index.js`：
```js
import { useUserStore } from '@/store'

const { token, userInfo, login, logout } = useUserStore()
```

## 三、三个页面规格

### 3.1 登录页（`/login`）

**文件**：`src/views/Login.jsx`

**功能**：
- 用户名 + 密码表单（Ant Design Form）
- 调用 `POST /api/v1/auth/login`
- 成功：token 存 localStorage，跳转 `/dashboard`
- 失败：显示错误提示

**视觉**：美食主题暗色风，深色背景，居中卡片

**关键代码逻辑**：
```js
const onFinish = async (values) => {
  const res = await login({ username: values.username, password: values.password })
  if (res.code === 200) {
    localStorage.setItem('token', res.data.access_token)
    useUserStore.getState().setToken(res.data.access_token)
    navigate('/dashboard')
  } else {
    message.error(res.message)
  }
}
```

## 四、ECharts 使用规范

### 安装
```bash
npm install echarts echarts-for-react echarts-wordcloud
```

### 上海地图（必须先注册）
```bash
# 下载地图文件，放到 frontend/public/shanghai.json
# 下载地址：https://geo.datav.aliyun.com/areas_v3/bound/310000_full.json
```

```jsx
import ReactECharts from 'echarts-for-react'
import * as echarts from 'echarts'
import shanghaiJson from '/shanghai.json'  // 注意是 /public 下的文件

// 在组件外注册一次
echarts.registerMap('shanghai', shanghaiJson)

// 使用
const option = {
  series: [{
    type: 'map',
    map: 'shanghai',
    data: [{ name: '浦东新区', value: 450 }, ...]
  }]
}

<ReactECharts option={option} style={{ height: '400px' }} />
```

### 词云图
```jsx
import ReactECharts from 'echarts-for-react'
```

## 五、路由与权限

### 路由配置（`src/router/index.jsx`）
```jsx
import { createBrowserRouter } from 'react-router-dom'

const router = createBrowserRouter([
  { path: '/login',     element: <Login /> },
  { path: '/dashboard', element: <ProtectedRoute><Dashboard /></ProtectedRoute> },
  { path: '/search',    element: <ProtectedRoute><Search /></ProtectedRoute> },
  { path: '/',          element: <Navigate to="/dashboard" /> },
])
```

### 路由守卫（`ProtectedRoute`）
```jsx
function ProtectedRoute({ children }) {
  const token = localStorage.getItem('token')
  if (!token) return <Navigate to="/login" replace />
  return children
}
```

## 八、常见问题

**图表不显示**
→ 检查父容器有没有设置高度，ECharts 需要明确的高度

**上海地图不显示**
→ 确认 `public/shanghai.json` 存在，且 `echarts.registerMap` 在 `<ReactECharts>` 渲染前执行

**接口 404**
→ 检查 `vite.config.js` 的代理配置，确认后端在 8000 端口运行

**CORS 报错**
→ 后端没加 CORS 中间件，让成员 B 在 `backend/app/main.py` 加上

**登录后刷新页面丢失登录状态**
→ Zustand store 初始化时从 localStorage 读取 token：
```js
const useUserStore = create((set) => ({
  token: localStorage.getItem('token') || null,
  // ...
}))
```
