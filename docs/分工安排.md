## 成员 B — 后端开发 1（框架 + 认证 + 搜索）

**Git 分支**：`feature/backend-core`

**负责目录**：`backend/app/core/`、`backend/app/api/v1/auth.py`、`backend/app/api/v1/restaurants.py`

### 第一阶段（第 1-3 天）

- [ ] 配置 FastAPI 项目结构，确认 `uvicorn` 正常启动
- [ ] 配置数据库连接（`core/config.py` 读取 `.env`）
- [ ] 配置 CORS 中间件（允许前端跨域）
- [ ] 实现 SQLAlchemy 模型（`models/restaurant.py`、`models/user.py`）
- [ ] 实现认证模块（`core/security.py`）
  - BCrypt 密码哈希
  - JWT token 生成和验证
  - 依赖注入 `get_current_user`

### 第二阶段（第 4-6 天）

- [ ] 实现登录接口（`api/v1/auth.py`）
  - `POST /api/v1/auth/login`
  - 验证用户名密码，返回 JWT token
- [ ] 实现餐厅搜索接口（`api/v1/restaurants.py`）
  - `GET /api/v1/restaurants/search`（支持关键词、菜系、区域、价格、评分筛选，分页）
  - `GET /api/v1/restaurants/{id}`（餐厅详情）
- [ ] 编写 Pydantic schemas（`schemas/restaurant.py`、`schemas/auth.py`）

### 第三阶段（第 7-8 天）

- [ ] 与前端联调，确认接口返回格式正确
- [ ] 处理边界情况（空结果、参数缺失等）

### 注意事项

- 统一响应格式：`{"code": 200, "message": "success", "data": ...}`
- 搜索接口需要支持模糊查询（`LIKE %keyword%`）
- 分页参数：`page`（从 1 开始）、`page_size`（默认 20）

**负责目录**：`crawler/shanghai_food/spiders/meituan_spider.py`、`crawler/shanghai_food/pipelines.py`

### 第二阶段（第 3-6 天）

- [ ] 实现美团爬虫（`meituan_spider.py`）
  - 爬取上海各区餐厅列表

- [ ] 实现数据管道（`pipelines.py`）

- 爬虫日志保存到 `crawler/logs/` 目录
- 不要爬取用户隐私数据（用户 ID、手机号等）

## 协作规范

### 每天同步

每天晚上在群里同步：
1. 今天完成了什么
2. 明天计划做什么
3. 遇到什么问题需要帮助

### 接口联调

- 后端完成一个接口后，在群里通知前端
- 前端把对应接口的 `USE_MOCK` 改为 `false` 进行测试
- 发现问题在群里说，不要自己改别人的代码

### 紧急情况

- 爬虫失败 → 成员 E 立即切换备用方案，不要等
- 接口报错 → 先看 `http://localhost:8000/docs` 测试接口是否正常
- 环境问题 → 参考开发指南，还是不行就在群里问
